fix: OrpheusDL settings opslaan (modules ipv module_settings) + zoekbalk debounce

Bug 1: Alle settingsKey-waarden in PLATFORM_DEFS gebruikten verkeerd pad
'module_settings.X' i.p.v. 'modules.X'. Hierdoor schreef de frontend
credentials naar settings.module_settings.qobuz terwijl OrpheusDL leest
van settings.modules.qobuz — die bleef dus leeg na opslaan.

Fix:
- public/src/components/orpheusSettings.js: settingsKey voor alle 9 platforms
  gewijzigd van module_settings.X naar modules.X
- services/orpheus.js: loadDefaultSettings() genereert nu ook modules:{} i.p.v.
  module_settings:{} zodat nieuw gegenereerde settings.json consistent is

Bug 2: Zoekbalk in OrpheusDL-view reageerde niet op typen, alleen op Enter
en klik op zoekknop.

Fix:
- public/src/views/orpheus.js: debounced input-listener (320ms) toegevoegd
  aan #oph-search-input, vergelijkbaar met andere views in de app

Bug 3: Qobuz downloads mislukten met 401 (User authentication required)

OrpheusDL gebruikte use_id_token: false waardoor elke download opnieuw
authenticeerde via app_id/app_secret. Die publieke app credentials gaven
een 401 van de Qobuz API.

Fix:
- data/orpheusdl/config/settings.json: use_id_token gewijzigd naar true
  zodat OrpheusDL een persistent user token gebruikt i.p.v. elke keer
  opnieuw via app credentials te authenticeren.
  (dit bestand is gitignored, change persists via Docker volume mount)

Rebuild: npm run build:frontend

Uitvoeren:
  git add public/src/components/orpheusSettings.js public/src/views/orpheus.js \
          services/orpheus.js public/app.js public/chunks/
  git commit -F COMMIT_MESSAGE.txt
